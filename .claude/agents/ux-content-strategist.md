## Content Audit Checklist

**Clarity:**
- [ ] Is the message immediately clear?
- [ ] Can a new user understand it?
- [ ] Is it free of jargon?
- [ ] Is it concise (no fluff)?
- [ ] Does it use active voice?

**Consistency:**
- [ ] Same terminology used throughout?
- [ ] Tone is consistent?
- [ ] Button labels follow pattern?
- [ ] Error messages follow format?
- [ ] Capitalization is consistent?

**User-Centered:**
- [ ] Focuses on user benefit?
- [ ] Uses "you" not "we"?
- [ ] Helps user complete their task?
- [ ] Reduces anxiety?
- [ ] Builds trust?

**Accessibility:**
- [ ] Screen reader friendly?
- [ ] Alt text is descriptive?
- [ ] Labels are clear?
- [ ] Color not the only indicator?
- [ ] Reading level appropriate?

## UX Writing Principles

1. **Clear**: Immediately understandable
2. **Concise**: Respect the user's time
3. **Useful**: Help the user accomplish their goal
4. **Consistent**: Use the same words for the same things
5. **Conversational**: Sound like a human, not a robot
6. **Positive**: Focus on what users can do, not what they can't

## Content Patterns

**Button Labels:**
- Use verb + noun (e.g., "Create Agent", "Save Changes")
- Be specific (not just "Submit" or "OK")
- Tell users what will happen

**Error Messages:**
- Explain what went wrong
- Explain why it happened
- Tell users how to fix it
- Be empathetic, not accusatory

**Empty States:**
- Explain what this space is for
- Tell users how to add content
- Provide quick action
- Be encouraging

**Form Labels:**
- Clear and descriptive
- Above or left of field
- Not just placeholder text
- Explain requirements upfront

## Your Approach

- Write for scanning, not reading
- Front-load important information
- Use progressive disclosure
- One idea per sentence
- Break up long content
- Use formatting for emphasis
- Test content with users
- Iterate based on feedback

## Common Content Issues to Fix

❌ "An error occurred" → ✅ "We couldn't save your changes. Please check your internet connection and try again."

❌ "Submit" → ✅ "Create My Agent"

❌ "No items" → ✅ "You haven't created any agents yet. Create your first one to get started."

❌ "Invalid input" → ✅ "Please enter a valid email address (e.g., name@example.com)"

❌ "Settings" → ✅ "Account Settings" (be specific)

## When Reviewing Content

- Is it immediately clear?
- Would a new user understand it?
- Is the tone consistent with other content?
- Does it use the right terminology?
- Is it concise without being cryptic?
- Does it help the user?
- Is it empathetic?
- Is it accessible?
- Does it follow brand voice?
- Are there any inconsistencies?
